# Inverted RSA

これは、RSA暗号がテーマになった問題です。

通常のRSAでは、異なる素数 `p, q` を使って

```python
n = p * q
phi = (p - 1) * (q - 1)
````

を計算します。

しかし、この問題では片方の素数が負になっています。

```python
p = -getPrime(384)
q = getPrime(384)
n = p * q
d = pow(e, -1, (p - 1) * (q - 1))
```

ここで `p` は負の数なので、`n` も負になります。

正の素数を `P = -p` とすると、本来のRSAで使うべき法は

```python
N = P * q
```

です。

つまり、与えられた `n` に対して

```python
N = -n
```

とすれば、通常の正のRSA法になります。

問題のポイントは、誤った `phi` で作られた秘密指数 `d` によって得られた `m2` です。

```python
m2 = pow(c, d, n)
```

この `d` は

```python
(p - 1) * (q - 1)
```

を使って作られていますが、`p` が負なのでこれは本来の

```python
(P - 1) * (q - 1)
```

とは異なります。

ただし、`q - 1` は共通して含まれているため、`m2` をもう一度 `e` 乗すると、少なくとも `mod q` では元の暗号文 `c` と一致します。

つまり、

```python
m2^e ≡ c (mod q)
```

となります。

そのため、

```python
gcd(m2^e - c, N)
```

を計算すると、共通因子として `q` が得られます。

```python
q = gcd(pow(m2, e, N) - c, N)
p = N // q
```

あとは通常のRSAと同じように、正しい `phi` を計算して復号します。

```python
phi = (p - 1) * (q - 1)
d = pow(e, -1, phi)
m = pow(c, d, N)
```
