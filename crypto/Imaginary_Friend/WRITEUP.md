# Imaginary Friend

## 暗号化手順の整理

この問題では、`GF(p)` 上に `i^2 = -1` を満たす元 `i` を追加した拡大体 `GF(p^2)` 上で計算を行っています。簡単に言い直すと、実部と虚部をそれぞれ `mod p` で計算するような複素数の世界での話です。

1. 大きさが `d × d` の行列 `M` をランダムに用意する。
2. 要素数 `d` のベクトル `state` をランダムに用意する。
3. PRNGを呼ぶたびに `state = M * state` と更新する。
4. 更新後の `state[0]` をPRNGの出力とする。
5. `flag` を6バイトずつ区切り、それぞれにPRNGの出力を加えたものを暗号文`C`とする。

## 未知数と方程式の整理

### 未知数

出力として与えられているものは、

- 行列`M`
- 暗号文`C=(c_0, c_1, ..., c_15)`

です。したがって、フラグを除けば、判明していないのは`state`のみになります。特に初期の`state`が分かれば、復号はできそうと分かります。つまり、未知数は`state`の要素数`9 * 2 = 18`です。これらの未知数を`state = (a_0 + b_0 * i, a_1 + b_1 * i, ..., a_8 + b_8 * i)`と置くことにしましょう。

### 方程式

まず、`flag`の暗号化の過程を見ると、`flag`の値は実部のみにしか反映されないことがわかります。つまり、`C`の虚部はPRNGの出力をそのまま反映していることがわかります。したがって、

```
Im(c_k) = Im((M^(k+1) * state)[0])
```

という式が成り立ちます。ここで、暗号ブロックは全部で16個あるので、方程式が16個導けました。

次に、`SECCON Beginners CTF 2026` 全体のルールとして与えられているフラグのフォーマットを見ます。すると、フラグは必ず `ctf4b{` から始まることが分かります。おあつらえ向きに、このクリブは6バイトです。つまり、平文の最初のブロックが分かっています。ここから以下の式が成り立ちます。

```python
Re(c_0) = bytes_to_long(b"ctf4b{") + Re((M * state)[0])
```

以上より合計で17個の方程式が導けました。

### 自由度1をどうするか

線形方程式だけでは、初期 `state` は一意には決まりません。

しかし、解空間は1次元なので、初期 `state` は

```
state = state_0 + t * v
```

という形で表せます。

ここで、`t` は `GF(p)` 上の1変数です。

この形を暗号化式に戻すと、各平文ブロックも `t` の一次式として表せます。

```
m_k = alpha_k + beta_k * t mod p
```

あとは、各 `m_k` が6バイトのprintable ASCIIであるという制約を使います。

```
0 <= m_k < 2^48
0x20 <= 各byte <= 0x7e
```
ここで `t` は `GF(p)` 上の値なので、候補は約 `2^56` 通りあります。

しかし、正しい `t` を代入した場合、各平文ブロック

```
m_k = alpha_k + beta_k * t mod p
```

は必ず6バイトの整数になります。

```
0 <= m_k < 2^48
```

さらに、flag は printable ASCII なので、6バイトに戻したときの各バイトは

```
0x20 <= byte <= 0x7e
```

を満たす必要があります。

つまり、t は全てのブロックについて「6バイトの printable ASCII になる」という条件を満たさなければなりません。

この条件は十分に強いため、CVPを用いて、残っていた1自由度を潰すことができます。

最終的に得られる `t` から全ての `m_k` を計算し、6バイトずつ結合すればflagを復元できます。
